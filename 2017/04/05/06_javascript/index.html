escape、encodeURI、encodeURIComponent区别

By Jinmengjie

2017-04-05 更新日期:2017-04-24

文章目錄 
    escape('http://localhost:4046/2017/04/05/06_javascript/index.html#id=1')
    "http%3A//localhost%3A4046/2017/04/05/06_javascript/index.html%23id%3D1"
    encodeURI('http://localhost:4046/2017/04/05/06_javascript/index.html#id=1')
    "http://localhost:4046/2017/04/05/06_javascript/index.html#id=1"
    encodeURIComponent('http://localhost:4046/2017/04/05/06_javascript/index.html#id=1')
    "http%3A%2F%2Flocalhost%3A4046%2F2017%2F04%2F05%2F06_javascript%2Findex.html%23id%3D1"

escape()对字符串编码，该方法不会对 ASCII 字母和数字进行编码，也不会对下面这些 ASCII 标点符号进行编码： - _ . ! ~ * ‘ ( ) 。其他所有的字符都会被转义序列替换。

encodeURI()该方法不会对 ASCII 字母和数字进行编码，也不会对这些 ASCII 标点符号进行编码： - _ . ! ~ * ‘ ( ) 。该方法的目的是对 URI 进行完整的编码，因此对以下在 URI 中具有特殊含义的 ASCII 标点符号，encodeURI() 函数是不会进行转义的：;/?:@&=+$,#

encodeURIComponent()该方法不会对 ASCII 字母和数字进行编码，也不会对这些 ASCII 标点符号进行编码： - _ . ! ~ * ‘ ( ) 。其他字符（比如 ：;/?:@&=+$,# 这些用于分隔 URI 组件的标点符号），都是由一个或多个十六进制的转义序列替换的。

请注意 encodeURIComponent() 函数 与 encodeURI() 函数的区别之处，前者假定它的参数是 URI 的一部分（比如协议、主机名、路径或查询字符串）。因此 encodeURIComponent() 函数将转义用于分隔 URI 各个部分的标点符号。
